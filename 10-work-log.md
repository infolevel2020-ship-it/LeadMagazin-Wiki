## РЕШЕНИЕ по резервному копированию (зафиксировано) [Я + Клод]

Резервное копирование решено через ХОСТИНГ reg.ru.
- reg.ru делает АВТОМАТИЧЕСКИЕ ЕЖЕДНЕВНЫЕ бэкапы (00:00-02:00 МСК): весь аккаунт, все файлы, файлы сайта leadmagazin.ru, все базы данных + БД u3276924_leadmagazin
- Хранится история за несколько дней (видны 09, 10, 11 июня)
- Доступно: восстановление, формирование архива, отправка ссылки на почту
- Раздел: ISPmanager → Резервные копии (backup243.hosting.reg.ru)
- Это надёжнее облака Битрикса (2 ГБ, мало)

Красная полоса в Битриксе ("ошибка создания резервной копии"): причина — облако Битрикса (2 ГБ) переполнено единственной копией 1.59 ГБ (79%), серверы облачного мониторинга Битрикса (IP 212.233.98.49) периодически дёргают /bitrix/tools/backup.php, копия не вмещается → ошибка.
План устранения: удалить копию из облака Битрикса (теперь безопасно — хостинг страхует ежедневно). Облачным бэкапом Битрикса больше не пользуемся.
