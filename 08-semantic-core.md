## Принципы построения СЯ [Клод + ДжПТ]

1. Сначала — структура и контент. Города — потом.
2. Слово "заявки" часто больше слова "лиды" по частотности — используем оба.
3. Люди не всегда знают слово "лид" — используем синонимы: заявки, клиенты, обращения, заказы.
4. Каждый кластер = отдельная страница с уникальным URL.
5. Сначала 20 ключевых страниц. Потом масштаб.

## ДВА БИЗНЕС-НАПРАВЛЕНИЯ — не путать [Клод + ДжПТ]

У сайта две разные бизнес-модели:

1. **Биржа лидов / Каталог** (`/catalog/`, `/lidy-dlya-.../`) — продажа ГОТОВЫХ заявок. Товар = лид (вкл. квал-лиды).
2. **Услуги / Лидогенерация** (`/services/...`) — УСЛУГА по организации потока лидов под заказ.

Формулировка: Каталог лидов = купить готовые лиды. Услуги = помочь организовать поток лидов.

⚠️ ГЛАВНОЕ (см. файл 13): обе модели — это ТОЧКИ ЗАХВАТА ЗАЯВОК, а не онлайн-покупки. Цель сайта = сбор заявок. После заявки работает менеджер. Каталог и услуги — витрины, подчинённые конверсии в заявку.

## РЕШЕНИЕ по "Купить лиды" [Клод + ДжПТ] — против каннибализации

Отказались от отдельных страниц вида `/kupit-lidy-dlya-nedvizhimosti/`. Причина: это тот же интент что `/lidy-na-nedvizhimost/` → две страницы под один запрос = каннибализация.

**Одна страница категории** `/lidy-na-nedvizhimost/` закрывает ВСЕ близкие коммерческие запросы ("купить лиды для недвижимости", "купить заявки на недвижимость", "купить клиентов для агентства") через Title, H1, контент, коммерческие блоки, перелинковку.

`/kupit-lidy/` — отдельная общая посадочная под запросы "купить лиды", "покупка лидов", "где купить лиды", "биржа лидов". НЕ родитель категорий, НЕ дубль ниш.

## РЕШЕНИЕ по лидогенерации [Клод + ДжПТ] — в /services/, не отдельная вертикаль

Отказались от отдельной большой вертикали `/lidogeneraciya/...`. Причины: пересечение с каталогом, путаница для пользователя, "лидогенерация" менее массовый термин, риск что вырастет в набор услуг.

Лидогенерация развивается внутри `/services/` (подробные задачи — в файле 13-project-goals.md):
```
/services/lidogeneraciya/                       ← хаб
/services/lidogeneraciya-dlya-{ниша}/           ← по нишам (посадочные)
```
Услуги-сервисы (колл-центр, виджеты, реклама, контент-заводы и т.д.) — конкретные страницы определим при проработке. Направление будет расти по мере развития.
Примечание: квал-лиды (готовый лид = товар) идут в КАТАЛОГ, не в услуги.
