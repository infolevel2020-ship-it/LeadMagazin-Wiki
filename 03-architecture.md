## Целевая архитектура проекта [ДжПТ]

ЛидМагазин проектируется не как корпоративный сайт, а как будущий маркетплейс покупки и продажи лидов.

Текущий запуск выполняется на связке Битрикс + Аспро для быстрого выхода на рынок.

Архитектура должна позволять:

* запуск проекта без больших вложений;
* минимизировать зависимость от Аспро;
* сохранить SEO и контентную часть сайта;
* в будущем перенести бизнес-логику на Laravel;
* масштабировать платформу без полного переписывания сайта.

---

## Архитектурные принципы [ДжПТ]

### 1. Разделение контента и бизнес-логики

Контент:

* статьи;
* категории;
* SEO-страницы;
* посадочные страницы.

Бизнес-логика:

* лиды;
* сделки;
* платежи;
* рейтинги;
* личные кабинеты.

Эти части должны быть максимально независимы друг от друга.

### 2. API First

Любая новая сущность проектируется с учетом будущего API.

Даже если API отсутствует на текущем этапе.

### 3. Минимизация зависимости от Аспро

Запрещается создавать критически важный функционал внутри шаблона Аспро.

### 4. Минимизация зависимости от Битрикс

Битрикс используется как CMS и SEO-платформа.

Бизнес-ядро должно иметь возможность переноса в Laravel.

### 5. Модульность

Каждый новый функциональный блок должен иметь возможность независимого развития.

## Enterprise фундамент — что закладываем сейчас [ДжПТ + Клод]

### ✅ Закладываем сейчас:

* Чистую структуру URL
* Нормальную модель данных
* Разделение контента и бизнес-логики
* Возможность вынести биржу на отдельный сервис
* API-first мышление (даже если API пока нет)

### ❌ НЕ закладываем сейчас:

* Микросервисы
* Kubernetes
* Сложную DevOps-инфраструктуру
* Десятки серверов
* Преждевременную оптимизацию
